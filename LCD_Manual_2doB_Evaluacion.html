🖥️SECCIÓN 1 — ¿Qué hace este proyecto?
Este proyecto hace algo muy especial: controla una pantalla LCD de 16×2 caracteres sin usar ninguna librería — todo a mano, bit a bit.

Normalmente usamos LiquidCrystal.h y la librería se encarga de todo. Aquí el alumno debe entender cómo funciona la pantalla por dentro: cómo enviarle comandos para limpiarla, encenderla, posicionar el cursor, y cómo enviarle los datos de cada carácter.

El resultado visible es que la pantalla muestra la palabra "HII" en un bucle infinito — pero lo interesante no es lo que muestra, sino cómo lo logra: manipulando directamente los registros de puerto del Arduino (PORTB, PORTC, PORTD) en lugar de usar digitalWrite(). El circuito también incluye un potenciómetro para ajustar el contraste, un multímetro para medir voltajes, LEDs y un botón. 
// Lo que hace el loop: mostrar "HII" para siempre
while(1) {
  command(CLEAR_1); command(CLEAR_2);  // limpiar pantalla
  ch(H_1); ch(H_2);                   // carácter 'H'
  ch(I_1); ch(I_2);                   // carácter 'I'
  ch(I_1); ch(I_2);                   // carácter 'I'
}
⚡SECCIÓN 2 — Registros de puerto: PORTB, PORTC, PORTD
Normalmente usamos digitalWrite(pin, HIGH) para controlar un pin. Pero hay una forma más directa y rápida: escribir en el registro de puerto completo de una sola vez.

El Arduino UNO tiene tres grupos de pines físicos llamados puertos:
• PORTD → pines digitales 0 al 7
• PORTB → pines digitales 8 al 13
• PORTC → pines analógicos A0 al A5

Cuando el código escribe PORTD = command, está configurando los 8 pines del puerto D al mismo tiempo, en lugar de hacerlo uno por uno. El prefijo DDR (Data Direction Register) define si cada pin es entrada o salida: DDRD = B11111111 pone todos los pines del puerto D como salidas de golpe. 
// Configurar TODOS los pines de cada puerto como salidas
DDRD = B11111111; // pines 0-7  → todos OUTPUT
DDRB = B11111111; // pines 8-13 → todos OUTPUT
DDRC = B11111111; // pines A0-A5 → todos OUTPUT

// Escribir en el puerto D de una sola instrucción:
PORTD = 0b0010; // pin 1 = HIGH, el resto = LOW
// equivale a: digitalWrite(1, HIGH); y el resto LOW
🔢SECCIÓN 3 — Modo 4-bit, #define y binario
El LCD puede recibir datos de dos formas: en modo 8-bit (8 cables de datos) o en modo 4-bit (4 cables de datos). El modo 4-bit usa menos pines pero envía cada instrucción en dos partes (nibbles): primero los 4 bits altos, luego los 4 bits bajos. Por eso siempre vemos las funciones command() y ch() llamadas en pares.

Los #define al inicio del código no son variables — son sustituciones de texto que hace el compilador antes de compilar. #define CLEAR_1 0b0000 significa que cada vez que el código diga CLEAR_1, el compilador lo reemplaza con 0b0000 (cero en binario). Esto hace el código más legible sin usar memoria extra.

El prefijo 0b indica que el número está en base binaria: 0b0010 = 2 en decimal. Cada posición es una potencia de 2 de derecha a izquierda. 
// #define: nombre → valor constante (el compilador hace la sustitución)
#define DATAMODE 0b0010 // modo 4-bit = 2 decimal
#define CLEAR_1  0b0000 // primer nibble del comando CLEAR
#define CLEAR_2  0b0001 // segundo nibble del comando CLEAR
#define H_1      0b0100 // nibble alto de 'H' (ASCII 0x48)
#define H_2      0b1000 // nibble bajo de 'H'

// Binario rápido:
// 0b0001 = 1  |  0b0010 = 2  |  0b0100 = 4  |  0b1000 = 8
📡SECCIÓN 4 — Las funciones command() y ch()
El proyecto tiene dos funciones personalizadas que se encargan de comunicarse con el LCD:

command(int c): Envía un comando al LCD (limpiar, encender, posicionar cursor). Pone PORTC = B0 (RS = 0, modo comando), escribe el nibble en PORTD, y genera un pulso en PORTB (Enable: sube a B1, espera, baja a B0). El LCD lee el dato en el flanco de bajada del Enable.

ch(int c): Envía un carácter al LCD para que lo muestre. La única diferencia con command() es que pone PORTC = B1 (RS = 1, modo dato). El pin RS (Register Select) le dice al LCD: "esto es un dato a mostrar, no una instrucción".

El delay(DELAY) dentro de ambas funciones (donde DELAY = 2 ms) le da tiempo al LCD para procesar cada nibble antes de recibir el siguiente. 
void command(int c) {
  PORTC = B0;   // RS = 0 → modo COMANDO
  PORTD = c;    // datos en el bus
  PORTB = B1;   // Enable → HIGH (pulso)
  delay(DELAY); // esperar 2 ms
  PORTB = B0;   // Enable → LOW (LCD procesa)
}
void ch(int c) {
  PORTD = c;
  PORTC = B1;   // RS = 1 → modo DATO (carácter)
  PORTB = B1;
  delay(DELAY);
  PORTB = B0;
}
