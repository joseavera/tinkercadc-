🔌SECCIÓN 1 — Librerías, pines y variables globales
El código comienza incluyendo dos librerías: LiquidCrystal.h para controlar la pantalla LCD de 16×2 caracteres, y Adafruit_NeoPixel.h para manejar los LEDs de color.

Después se definen los pines de cada botón y componente usando #define, que es como ponerle un nombre a un número para no tener que recordarlo. Por ejemplo, #define P1U 6 significa que el botón "jugador 1 arriba" está en el pin 6.

Luego se crean las variables globales: arreglos byte que representan las paletas y la pelota como matrices de 16 posiciones, y variables int y bool que guardan la puntuación, la dirección y si el juego está corriendo. 
// Incluir librería
#include <LiquidCrystal.h>
LiquidCrystal lcd(12, 11, 5, 4, 3, 2);
#define P1U 6   // botón jugador 1 arriba
#define piezo 10
int score1 = 0, score2 = 0;
boolean game = 0;
char direction = 'L';
⚙️SECCIÓN 2 — setup() y el bucle de espera
La función setup() se ejecuta una sola vez al encender el Arduino. Aquí se inicializan los LEDs NeoPixel, se configuran los pines de los botones como entradas (INPUT) y el zumbador como salida (OUTPUT). También inicia la pantalla LCD y muestra el nombre del juego por 1.5 segundos.

Dentro de loop(), hay un bucle while que mantiene el programa esperando hasta que el jugador presione el botón de inicio. Se usa digitalRead(start) == HIGH para detectar cuándo se presiona. Una vez presionado, la variable game cambia a 1 y el juego comienza. 
void setup() {
  pinMode(P1U, INPUT);
  pinMode(piezo, OUTPUT);
  lcd.begin(16, 2);
  delay(1500);
}
while (game == 0) {
  if (digitalRead(start) == HIGH) {
    game = 1;
  }
}
🎮SECCIÓN 3 — Lógica del juego: paletas, pelota y puntuación
Ya en el juego, el programa lee los botones de cada jugador y llama funciones como paddle1Up() o paddle1Down() que mueven la paleta desplazando los valores en el arreglo.

La pelota avanza frame a frame: si va a la izquierda (direction == 'L'), la variable x decrece; si va a la derecha, aumenta. Cuando llega al extremo (posición x == 3 o x == 15), el programa verifica si la pelota coincide con la paleta. Si sí, cambia la dirección y suma un punto. Si no, muestra "Game over", enciende los LEDs y reinicia.

El bounce controla si la pelota se mueve hacia arriba o hacia abajo para darle efecto diagonal.
