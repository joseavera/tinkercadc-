🌬️SECCIÓN 1 — ¿Qué hace este proyecto?
Este proyecto simula un ventilador inteligente que toma decisiones automáticas. Hace tres cosas al mismo tiempo:

1. Detecta si hay alguien en el cuarto usando el sensor PIR. Si no hay nadie, el ventilador se apaga para ahorrar energía.
2. Mide la temperatura con el sensor TMP36 y decide a qué velocidad debe girar el motor — más calor = más velocidad.
3. Informa al usuario mostrando en la pantalla LCD la temperatura actual y el nivel de ventilación, o el mensaje "Sin presencia detectada" si no hay nadie.

El Arduino es el "cerebro": lee los sensores, calcula la velocidad correcta y manda la señal al motor a través del transistor TIP120. 
// Resumen del flujo principal
if (movimento == HIGH) {
  // hay alguien → encender ventilador
  analogWrite(motorPin, velocidade);
} else {
  // nadie → apagar ventilador
  analogWrite(motorPin, 0);
}
🔩SECCIÓN 2 — Los componentes clave
TMP36 — Sensor de temperatura: Es un pequeño chip de 3 pines que genera un voltaje proporcional a la temperatura. El Arduino lo lee con analogRead() en el pin A0. La lectura va de 0 a 1023, y el código la convierte a grados Celsius con la fórmula:
(lectura × 5.0 / 1024.0 − 0.5) × 100

TIP120 — Transistor Darlington: El Arduino no puede dar suficiente corriente para girar un motor directamente. El TIP120 actúa como un interruptor electrónico controlado: el Arduino manda una señal pequeña a su base (pin B), y el transistor deja pasar la corriente grande que necesita el motor entre el colector (C) y el emisor (E).

Diodo de protección: Cuando el motor se apaga, genera una corriente de rebote hacia atrás. El diodo actúa como una válvula de un solo sentido: deja pasar corriente en una dirección y bloquea ese rebote para proteger el TIP120 y el Arduino.
💻SECCIÓN 3 — Estructura del código
El programa tiene la estructura clásica Arduino:

setup(): Se ejecuta una sola vez. Configura los pines (INPUT para el PIR, OUTPUT para el motor), inicia la pantalla LCD y abre la comunicación serial.

loop(): Se repite indefinidamente. En cada ciclo: lee el PIR, lee el TMP36, calcula la temperatura, decide la velocidad con una cadena de if / else if, escribe en el motor con analogWrite() y actualiza el LCD.

La función analogWrite(pin, valor) es clave: acepta valores de 0 a 255 y genera una señal PWM (modulación por ancho de pulso) que controla cuánta energía llega al motor — 0 es apagado, 255 es máxima velocidad. 
// Escala de velocidades según temperatura
if      (temperatura >= 11 && temperatura <= 15) velocidade = 42,  nivel = 1;
else if (temperatura >= 16 && temperatura <= 20) velocidade = 85,  nivel = 2;
else if (temperatura >= 21 && temperatura <= 25) velocidade = 127, nivel = 3;
else if (temperatura >= 26 && temperatura <= 30) velocidade = 170, nivel = 4;
else if (temperatura >= 31 && temperatura <= 35) velocidade = 212, nivel = 5;
else if (temperatura >= 36 && temperatura <= 40) velocidade = 255, nivel = 6;
